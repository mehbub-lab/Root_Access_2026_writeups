# Eavesdropping

## 1. Challenge Overview
**Scenario:** Bob and Alice communicate via an encrypted TLS channel. Eve eavesdrops and finds a flaw: Alice assumes all messages are from Bob and lacks authentication for debug commands.
**Objective:** Find the flag by exploiting this vulnerability.

## 2. Methodology & Solution Steps

### Step 1: Preliminary Analysis
We started by examining the packet capture `Convo.pcapng` to understand the network traffic.

**Command:**
```bash
tshark -r Convo.pcapng -qz conv,tcp
```
**Findings:**
*   **Main Traffic:** Encrypted TLS on port **4433** (TCP Stream 11).
*   **Suspicious Traffic:** Unencrypted traffic on port **9001** (TCP Stream 9).

### Step 2: Vulnerability Discovery
We investigated the unexpected traffic on port 9001.

**Command:**
```bash
tshark -r Convo.pcapng -Y "tcp.port == 9001" -x
```
**Observation (Frame 8340):**
A plaintext command `DEBUG_DUMP_TLS` was sent to Alice. This indicates an unauthenticated administrative interface.

```
0000  ... 44 45 42 55 47 5f 44 55 4d 50 5f 54 4c 53   ...DEBUG_DUMP_TLS
```

### Step 3: Exploitation - Extracting TLS Secrets
Alice responded to the debug command with a full dump of the TLS session secrets. We extracted this data to decrypt the main conversation.

**Command:**
```bash
tshark -r Convo.pcapng -Y "frame.number == 8342" -T fields -e tcp.payload | xxd -r -p > tls_secrets.txt
```

### Step 4: Traffic Decryption
Using the recovered secrets, we decrypted the TLS stream on port 4433.

**Command:**
```bash
tshark -r Convo.pcapng -o "tls.keylog_file:tls_secrets.txt" -q -z follow,tls,ascii,11
```
